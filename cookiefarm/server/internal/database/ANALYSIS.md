# Database Package — Reverse Engineering & TDD Analysis

## 1. Package Overview

**Module:** `server/database`
**Language:** Go
**Purpose:** Sole data-access layer of the server. Owns the SQLite schema, all SQL queries, the connection pool, a write-buffer (FlagCollector), and the mapping helpers that convert between domain objects and DB params.

---

## 2. Component Inventory

### 2.1 `connection.go` — DB factory

| Symbol | Kind | Responsibility |
|--------|------|----------------|
| `Config` | struct | Holds every knob needed to open and tune the connection pool (DSN, pool sizes, timeouts). |
| `NewDB(cfg Config) (*sql.DB, error)` | func | Opens a SQLite connection, applies pool settings, pings within 5 s, runs the embedded schema DDL, returns the ready `*sql.DB`. |
| `schemaSQL` | `//go:embed schema.sql` | The DDL is baked into the binary at compile time so the schema is always applied on startup. |

**Interface summary**

| | Detail |
|-|--------|
| Input | `Config` — all fields optional at the type level, but `DSN` must be a valid SQLite path/URI |
| Output | `(*sql.DB, error)` — non-nil `error` if dial, ping, or schema fails |
| Side effects | Creates/migrates the SQLite file on disk |

### 2.2 `models.go` — Domain model (generated by sqlc)

| Symbol | Kind | Responsibility |
|--------|------|----------------|
| `Flag` | struct | In-memory representation of a row in the `flags` table. Maps 1-to-1 to the DB schema. |

**Fields**

| Field | Type | Notes |
|-------|------|-------|
| `FlagCode` | `string` | Primary key — unique flag string |
| `ServiceName` | `string` | Attacked service name |
| `PortService` | `uint16` | Port of the service |
| `SubmitTime` | `uint64` | Unix timestamp: when the flag was queued |
| `ResponseTime` | `uint64` | Unix timestamp: when the scorer replied |
| `Msg` | `string` | Human-readable message from the scorer |
| `Status` | `string` | One of `UNSUBMITTED`, `ACCEPTED`, `DENIED`, `ERROR` |
| `TeamID` | `uint16` | Numeric team identifier |
| `Username` | `string` | Attacker username |
| `ExploitName` | `string` | Exploit/script that found the flag |

### 2.3 `db.go` — Query runner (generated by sqlc)

| Symbol | Kind | Responsibility |
|--------|------|----------------|

### 2.4 `query.sql.go` — All SQL operations (generated by sqlc)

Every method lives on `*Queries`.

| Method | SQL verb | Input params | Return | What it does |
|--------|----------|-------------|--------|--------------|
| `AddFlag(ctx, AddFlagParams) error` | `INSERT OR IGNORE` | All flag fields | `error` | Inserts a new flag; silently ignores duplicates (PK conflict). |
| `GetFlagByCode(ctx, flagCode string) (Flag, error)` | `SELECT` | `flag_code` string | `Flag, error` | Fetches a single flag by its primary key. |
| `GetAllFlags(ctx) ([]Flag, error)` | `SELECT` | — | `[]Flag, error` | Returns every flag, newest first. |
| `GetAllFlagCodes(ctx) ([]string, error)` | `SELECT` | — | `[]string, error` | Returns every `flag_code` (lighter projection of the above). |
| `GetFirstNFlags(ctx, limit int64) ([]Flag, error)` | `SELECT … LIMIT ?` | `limit` | `[]Flag, error` | Returns the N most-recently submitted full flag rows. |
| `GetFirstNFlagCodes(ctx, limit int64) ([]string, error)` | `SELECT … LIMIT ?` | `limit` | `[]string, error` | Returns the N most-recently submitted flag codes only. |
| `GetPagedFlags(ctx, GetPagedFlagsParams) ([]Flag, error)` | `SELECT … LIMIT ? OFFSET ?` | `Limit`, `Offset` | `[]Flag, error` | Paginated full-row fetch, newest first. |
| `GetPagedFlagCodes(ctx, GetPagedFlagCodesParams) ([]string, error)` | `SELECT … LIMIT ? OFFSET ?` | `Limit`, `Offset` | `[]string, error` | Paginated flag-code-only fetch. |
| `GetFlagsByTeam(ctx, GetFlagsByTeamParams) ([]Flag, error)` | `SELECT … WHERE team_id = ?` | `TeamID`, `Limit`, `Offset` | `[]Flag, error` | Paginated fetch scoped to a single team, newest first. |
| `GetUnsubmittedFlags(ctx, limit int64) ([]Flag, error)` | `SELECT … WHERE status = 'UNSUBMITTED'` | `limit` | `[]Flag, error` | Returns flags not yet submitted to the scorer, oldest first. |
| `GetUnsubmittedFlagCodes(ctx, limit int64) ([]string, error)` | `SELECT … WHERE status = 'UNSUBMITTED'` | `limit` | `[]string, error` | Same as above but code-only projection. |
| `GetFilteredFlags(ctx, GetFilteredFlagsParams) ([]Flag, error)` | `SELECT … WHERE …` | `TeamID`, `Status`, `ServiceName`, `Search`, `SearchField`, `SearchLike`, `Limit`, `Offset` | `[]Flag, error` | Multi-criteria filtered+paginated query. |
| `CountFilteredFlags(ctx, CountFilteredFlagsParams) (int64, error)` | `SELECT COUNT(*)` | Same as `GetFilteredFlags` | `int64, error` | Total count for the same multi-criteria filter (used for pagination metadata). |
| `CountFlags(ctx) (int64, error)` | `SELECT COUNT(*)` | — | `int64, error` | Total number of flags in the table. |
| `UpdateFlagStatusByCode(ctx, UpdateFlagStatusByCodeParams) error` | `UPDATE` | `FlagCode`, `Status`, `Msg`, `ResponseTime` | `error` | Updates status, scorer message, and response timestamp for one flag. |
| `DeleteFlagByCode(ctx, flagCode string) error` | `DELETE` | `flag_code` string | `error` | Deletes a single flag by PK. |
| `DeleteFlagByTTL(ctx, strftime interface{}) (int64, error)` | `DELETE … WHERE response_time < strftime(…)` | SQLite strftime modifier string (e.g. `"-7 days"`) | `int64, error` | Bulk-deletes flags older than the given age; returns the number of deleted rows. |

### 2.5 `store.go` — Transaction wrapper

| Symbol | Kind | Responsibility |
|--------|------|----------------|
| `Store` | struct | Composes `*sql.DB` + `*Queries`. Single choke-point for the rest of the server. |
| `NewStore(db *sql.DB) *Store` | func | Constructor — wraps an open `*sql.DB`. |
| `(s *Store) WithTx(ctx, fn func(*Queries) error) error` | method | Runs `fn` inside a transaction; auto-commits on success, auto-rolls back on error. |

### 2.6 `flag_collector.go` — Async write buffer

| Symbol | Kind | Responsibility |
|--------|------|----------------|
| `CollectorStats` | struct | Bookkeeping counters: total received, flushed, errors, timestamps. |
| `FlagCollector` | struct | The main actor. Buffers incoming `Flag` values and writes them to the DB in batches. |
| `GetCollector() *FlagCollector` | func | **Singleton** accessor (via `sync.Once`). Returns the one global `FlagCollector`. |
| `(fc) SetStore(s *Store)` | method | Injects the `*Store` dependency. Must be called before `Start`. |
| `(fc) Start()` | method | Launches the background timer goroutine. Idempotent — second call is a no-op. |
| `(fc) Stop() error` | method | Signals the goroutine to exit, triggers a final flush, logs summary stats. Returns the flush error if any. |
| `(fc) AddFlag(flag Flag) error` | method | Thread-safe. Appends to buffer; auto-starts if not running; triggers immediate flush if buffer reaches `maxBufferSize` (100). |
| `(fc) Flush() error` | method | Synchronous drain with `context.Background()`. |
| `(fc) FlushWithContext(ctx, ...) error` | method | Core drain: copies buffer out, calls `Queries.AddFlag` for each flag, re-queues on error (if space allows), broadcasts cond. |
| `(fc) GetBufferSize() int` | method | Thread-safe read of current buffer length. |
| `(fc) GetStats() CollectorStats` | method | Thread-safe snapshot of statistics. |
| `(fc) IsRunning() bool` | method | Thread-safe status check. |
| `updateFlushStats(err, flagCount)` | private | Updates `CollectorStats` after every flush attempt. |

**Constants**

| Constant | Value | Meaning |
|----------|-------|---------|
| `maxBufferSize` | 100 | Triggers an immediate flush when buffer reaches this size |
| `flushInterval` | 10 s | How often the timer goroutine flushes automatically |
| `flushTimeout` | 10 s | Context deadline applied to each timed flush |

## 3. Internal Dependency Graph

```

  NewDB()                                               │
         └--► schema.sql (embedded)                     │
                                                        ▼
                                                  store.go
db.go ------------------------------------------► Store.Queries (*Queries)
  Queries / DBTX                                        │
         │                                              │
         ▼                                              │
  query.sql.go                                          │
  (all SQL methods)                                     │
         ▲                                              │
         └----------------------------------------------┘
                                                        ▲
flag_collector.go --------------------------------------┘
  FlagCollector.store (*Store)

         ▲
         │  uses
  mapper.go
  MapFromFlagToDBParams()
```

| `protocols` (internal pkg) | `mapper.go` | `ResponseProtocol` type from the shared protocol package |

## 5. Dependency Impact Matrix

> "If I change X, what breaks?"

| Changed component | Directly affected | Notes |
|-------------------|------------------|-------|
| `schema.sql` | `connection.go`, every query in `query.sql.go` | Schema changes ripple through every query |
| `Flag` (models.go) | `mapper.go`, `flag_collector.go`, `query.sql.go` | Changing a field type/name is a multi-file change |
| `AddFlagParams` | `mapper.go`, `query.sql.go`, `flag_collector.go` | Regenerated by sqlc; mappers must stay in sync |
| `UpdateFlagStatusByCodeParams` | `mapper.go`, `query.sql.go` | Regenerated by sqlc |
| `Store.WithTx` | Any caller (currently `core` package) | Signature change breaks all transaction callers |
| `FlagCollector.AddFlag` | `core/flag_loop.go` (inferred) | The only ingest point for new flags from the network |
| `GetCollector()` | Any package that calls it | Singleton; hard to swap out — see Issue 4.2 |
| `MapFromResponseProtocolToParamsToUpdate` | `core/flag_loop.go` | Called when scorer responses are processed |
| `protocols.ResponseProtocol` | `mapper.go` | External package; changes require mapper update |

## 5. TDD Plan

## 6 Category Partitioning

#### `NewDB`

| Category | Partition | Expected outcome |
|----------|-----------|-----------------|
| DSN | valid in-memory (`:memory:`) | success, `*sql.DB` non-nil |
| DSN | valid file path | success |
| DSN | invalid / empty string | error on `Open` or `Ping` |
| Schema | valid embedded SQL | table `flags` exists after call |
| Pool config | zero values | no panic; defaults applied |

#### `Queries.AddFlag`

| Category | Partition | Expected |
|----------|-----------|---------|
| FlagCode | new unique value | row inserted, no error |
| FlagCode | duplicate of existing | silently ignored (`INSERT OR IGNORE`), no error |
| FlagCode | empty string | inserted (PK allows empty, schema doesn't forbid) |
| Status | any non-empty string | stored as-is |
| Numeric fields | zero values | inserted |

#### `Queries.GetFlagByCode`

| Category | Partition | Expected |
|----------|-----------|---------|
| FlagCode | exists in DB | returns correct `Flag`, no error |
| FlagCode | does not exist | returns `sql.ErrNoRows` |
| FlagCode | empty string | returns `sql.ErrNoRows` (unless empty-string flag was inserted) |

#### `Queries.UpdateFlagStatusByCode`

| Category | Partition | Expected |
|----------|-----------|---------|
| FlagCode | exists | status/msg/response_time updated |
| FlagCode | does not exist | no error (UPDATE affects 0 rows — acceptable) |
| Status | any string | stored as-is |
| ResponseTime | zero | stored as zero |
| ResponseTime | valid Unix timestamp | stored correctly |

#### `Queries.DeleteFlagByCode`

| Category | Partition | Expected |
|----------|-----------|---------|
| FlagCode | exists | row removed, no error |
| FlagCode | does not exist | no error (DELETE affects 0 rows) |

#### `Queries.DeleteFlagByTTL`

| Category | Partition | Expected |
|----------|-----------|---------|
| Modifier | `"-0 seconds"` | deletes flags with `response_time < now` |
| Modifier | very negative offset | deletes nothing |
| Modifier | very positive offset | may delete all old flags |
| DB | empty table | returns 0, no error |

#### `Queries.GetFilteredFlags` + `CountFilteredFlags`

| Category | Partition | Expected |
|----------|-----------|---------|
| TeamID | valid team with rows | only that team's rows |
| TeamID | team with no rows | empty slice |
| Status filter | "ACCEPTED" | only accepted rows |
| Search + SearchField | "flag_code" + partial code | only matching rows |
| Search + SearchField | "all" + term | matches across all searchable columns |
| Pagination | Limit=1, Offset=0 | exactly 1 row |
| Pagination | Offset > total rows | empty slice |

#### `Store.WithTx`

| Category | Partition | Expected |
|----------|-----------|---------|
| fn | returns nil | transaction committed |
| fn | returns error | transaction rolled back, error propagated |
| fn | panics | (not tested — Go convention) |

#### `FlagCollector.AddFlag`

| Category | Partition | Expected |
|----------|-----------|---------|
| Collector state | not started | auto-starts, flag buffered |
| Collector state | already started | flag buffered |
| Buffer | below `maxBufferSize` | no flush triggered |
| Buffer | reaches `maxBufferSize` | immediate flush triggered, buffer drained |
| Store | nil at flush time | error returned, flags re-queued |
| Flag | valid | buffered with no error |

#### `FlagCollector.FlushWithContext`

| Category | Partition | Expected |
|----------|-----------|---------|
| Buffer | empty | returns nil immediately |
| Buffer | 1 flag, store set | flag written to DB, buffer empty |
| Buffer | N flags, store set | all flags written, buffer empty |
| Store | nil | error returned, flags restored to buffer |
| Context | already cancelled | DB error, flags re-queued |

#### `MapFromFlagToDBParams`

| Category | Partition | Expected |
|----------|-----------|---------|
| All fields | fully populated Flag | 1-to-1 copy into AddFlagParams |
| All fields | zero-value Flag | zero-value AddFlagParams |

#### `MapFromResponseProtocolToParamsToUpdate`

| Category | Partition | Expected |
|----------|-----------|---------|
| All fields | fully populated ResponseProtocol | Flag/Status/Msg mapped; ResponseTime ≈ `time.Now().Unix()` (Issue 4.5 fixed) |
| All fields | empty ResponseProtocol | empty FlagCode/Status/Msg; ResponseTime ≈ `time.Now().Unix()` |

#### `FlagCollector.AddFlag` — mutex-leak path (Issue 4.4)

| Category | Partition | Expected |
|----------|-----------|---------|
| Buffer exactly at `maxBufferSize`, DB write fails | error path | error returned **and mutex released** — next `AddFlag` call must not deadlock |

#### `Stop()` — stats data race (Issue 4.8)

| Category | Partition | Expected |
|----------|-----------|---------|
| Stop called while timer goroutine is also flushing | concurrent access | no data race detected by `-race` |
