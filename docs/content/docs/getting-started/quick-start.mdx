# Quick Start

This page walks through three short steps:

1. Start the server (Docker Compose).
2. Install and authenticate the `ckc` CLI.
3. Create or run your first exploit and verify results in the dashboard.

---

## 1) Start the server

After the installation (either automatic or manual), you should have a `docker-compose.yml` file in your `cookiefarm` directory (or the root in case of an automatic remote installation). Start the server with:

```bash
docker compose up -d --build
```

Notes:
- If you prefer `docker-compose` (older CLI), replace `docker compose` with `docker-compose`.
- Wait a few seconds for services to initialize. Check logs with `docker compose logs -f`.

Verify the HTTP endpoints:

- Server API (default): http://localhost:8080/api/v1/
- Dashboard / Web UI (default): http://localhost:8080

---

## 2) Install and authenticate the `ckc` CLI

Install `ckc` into a virtualenv or with pipx:

```bash
# Example: install with pip
python -m venv .venv
. .venv/bin/activate
pip install --upgrade pip
pip install cookiefarm
```

Authenticate the CLI against your running server:

```bash
ckc config reset
ckc config login -P <PASSWORD> -H <SERVER_HOST> -p <SERVER_PORT> -u <USERNAME>
```


---

## 3) Run your first exploit

CookieFarm supports multiple exploit authoring approaches (Python runners, shell templates, etc.). The easiest is to use a simple built-in example or template bundled with the project.

List available exploits:

```bash
ckc exploit list
```

Probably if is the first time you run it, the list is empty.

Create a new exploit from the template:

```bash
# Create a new exploit skeleton from a template (example name: "sample-exploit")
ckc exploit create --name sample-exploit 
# This creates a new file called sample-exploit.py in ~/.config/cookiefarm/exploits/ with a basic template you can edit.
```

Run the exploit against a target (replace `<SERVICE_NAME>` with the name of the service to attack, as defined in your server config):

```bash
ckc exploit run -e sample-exploit -t 30 -T 5 -n <SERVICE_NAME>
```

Monitor progress:
- CLI output shows run progress and flag captures.
- Dashboard shows live exploit status, submissions, and flag history.

## Troubleshooting

- Containers fail to start:
  - Inspect logs: `docker compose logs -f`
  - Ensure required ports (8080, 3000) are free.
  - Confirm `.env` values are set (database URL, secret keys).

- Authentication fails:
  - Confirm server URL in `ckc config show`.
  - Check server logs for auth-related errors.

- Exploit runtime errors:
  - Re-run the exploit with debug logging: `ckc exploit run ... --debug`.
  - Check the server/exploit runner logs and dashboard details.
  - Try before in test mode with `ckc exploit test ...`
