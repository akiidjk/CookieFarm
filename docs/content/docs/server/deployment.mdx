# Server Deployment

This page describes recommended ways to deploy the CookieFarm server in production. It covers a pragmatic Docker Compose setup for small teams

---

## Quick Docker Compose example

In the repo, you can find two `docker-compose.yml` file one called `docker-compose.yml` for image pull from ghrc (no local image building needed) and `docker-compose.build.yml` for build the image in the local machine.

```yaml title="docker-compose.yml"
services:
  server:
    image: ghcr.io/bytethecookies/cookiefarm:latest
    restart: unless-stopped
    ports:
      - ${PORT}:${PORT}
    environment:
      PASSWORD: ${PASSWORD}
      CONFIG_FILE: ${CONFIG_FILE}
      PORT: ${PORT}
      DEBUG: ${DEBUG}
    env_file:
      - .env
    volumes:
      - sqlite_bp:/app/database
      - ./config.yml:/app/config.yml
    networks:
      - cookiefarm-network

volumes:
  sqlite_bp:
    driver: local

networks:
  cookiefarm-network:
    driver: bridge
    name: cookiefarm-network
```

```yaml title="docker-compose.build.yml"
services:
  server:
    build:
      context: .
      dockerfile: Dockerfile
      target: prod
    restart: unless-stopped
    ports:
      - ${PORT}:${PORT}
    environment:
      PASSWORD: ${PASSWORD}
      CONFIG_FILE: ${CONFIG_FILE}
      PORT: ${PORT}
      DEBUG: ${DEBUG}
    env_file:
      - .env
    volumes:
      - sqlite_bp:/app/database
      - ./config.yml:/app/config.yml
    networks:
      - cookiefarm-network

volumes:
  sqlite_bp:
    driver: local

networks:
  cookiefarm-network:
    driver: bridge
    name: cookiefarm-network
```

so for the first one do simple `docker compose up --build -d` for the second you have to specify the file like that `docker compose -f docker-compose.build.yml up -d`
