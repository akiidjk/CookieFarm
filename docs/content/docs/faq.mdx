## General

Q: What is CookieFarm?
A: CookieFarm is an exploit-orchestration framework for capture-the-flag (CTF) style competitions and automated offensive tooling. See the overview in [Getting Started](/docs/getting-started/index.mdx) and [Architecture](/docs/architecture/overview.mdx).

Q: Where can I find the project roadmap and release notes?
A: Roadmap and release notes are maintained in the repository's releases and project board. Check the repository homepage or the CHANGELOG in the repo root.

## Troubleshooting

Q: I'm seeing runtime errors when running exploits — where do I start debugging?
A: 1) Re-run the exploit with verbose/log flags (-v) where supported. 2) Check server logs and dashboard monitoring. 3) Confirm network connectivity and credentials. See general troubleshooting steps in the relevant section of each guide.

Q: Flags are not being submitted or validated — what could be wrong?
A: Verify the flag pipeline configuration in [architecture/flag-pipeline.mdx] and server API connectivity ([server/api-reference.mdx](/docs/server/api-reference.mdx)). Ensure submission credentials and rate limits are configured correctly.
