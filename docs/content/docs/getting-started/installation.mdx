This page covers the installation of CookieFarm's server and client components.

## Server Installation

Install this only if you are the sysadmin of the team only one server is necessary for the whole team. The server is responsible for managing flag, receiving it, and submitting it.

### Automatic Installation

```bash
curl -sSL cookiefarm.bytethecookies.org/install.sh | bash
```

The script will ask you the mode of installation if remote o build

#### Remote Installation

The script will download ONLY a docker-compose.yml and create .env and config.yml file, and then it will start the server with docker compose pulling the image from ghrc. This is the recommended way to install the server.

#### Build Installation

The script will clone the repository, create .env and config.yml file, and then it will start the server with docker compose building the image from the Dockerfile in the repository. This is recommended if you want to modify the server or if you want to run it in a local environment for testing.

### Manual Installation

1. Clone the repository and navigate to the server directory:
```bash
   git clone https://github.com/ByteTheCookies/CookieFarm.git
   cd CookieFarm
```

2. Create an `.env` file in the server directory to configure the environment settings:

```bash
cp .env.example .env
```

And edit the `.env` file to set your desired configuration.

```text
PORT=8080
PASSWORD=password
CONFIG_FILE=true
DEBUG=false
```

## Client Installation

The client is the component that the exploit developers will use to interact with the server, submit flags, and monitor results.

### Install with pip

```bash
python -m venv .venv
. .venv/bin/activate
pip install --upgrade pip
pip install cookiefarm
```

Try running `ckc --help` to verify the installation and see available commands.
