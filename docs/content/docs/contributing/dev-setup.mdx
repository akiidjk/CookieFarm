# Development setup — CookieFarm

This guide covers a typical local development setup you can use to contribute to CookieFarm. It contains recommended tool versions, quick start commands for each component (server, web/dashboard, Python client), testing and linting tips, and troubleshooting notes.

Adjust versions and commands to match the repository's `README.md`, `Makefile`, or component-specific README files.

---

## Prerequisites

Install the following on your machine:

- Git (latest)
- Go 1.20+ (or the version pinned by the repo)
- Node.js 18+ (LTS) and a package manager (npm, yarn, or pnpm)
- Python 3.10+ (or the version pinned by the repo)
- Docker & Docker Compose (for running services like databases, Redis, etc.)
- Recommended: Visual Studio Code with extensions for Go, TypeScript/React, Python, and Docker

Verify basics:

```bash title="commands.sh"
# check versions
git --version
go version
node --version
python --version
docker --version
docker compose version
```

## Environment files

If the project provides `.env.example` or similar, copy it and fill in secrets:

```bash title=".env.example"
# Example .env (copy to .env and edit)
# SERVER_BIND_ADDR=127.0.0.1:8080
# DATABASE_URL=postgres://user:pass@localhost:5432/cookiefarm
# REDIS_URL=redis://localhost:6379
# SECRET_KEY=replace-with-secure-value
```

Commands:

```bash title="commands.sh"
cp .env.example .env
# edit .env with your editor
```

If there is no `.env.example`, check repository docs for required environment variables.

## Backend (Go)

1. Install dependencies:

```bash title="commands.sh"
cd server   # or the repo's Go server path (check repository layout)
go mod download
```

2. Build or run:

```bash title="commands.sh"
# build everything
go build ./...
# run a server main package if present under cmd/
go run ./cmd/server
```

Note: Many Go projects keep CLI/server entrypoints under `./cmd/<name>`. Inspect the `cmd/` directory and use `go run ./cmd/<entry>` to run a specific service.

3. Tests:

```bash title="commands.sh"
go test ./... -v
```

4. Lint & format:

```bash title="commands.sh"
# formatting
gofmt -w .
# linting (if installed)
golangci-lint run ./...
```

## Frontend / Dashboard (TypeScript)

Locate the frontend directory (commonly named `frontend`, `web`, `dashboard`, or `ui`).

1. Install dependencies:

```bash title="commands.sh"
cd frontend      # replace with actual path
npm install      # or yarn install / pnpm install
```

2. Run development server:

```bash title="commands.sh"
npm run dev      # or npm start depending on the project
```

3. Build for production:

```bash title="commands.sh"
npm run build
```

4. Tests and lint:

```bash title="commands.sh"
npm test
npm run lint     # runs ESLint/TypeScript checks
```

## Python client (`ckc`)

The Python client often lives in `client/` and is installable as a package.

1. Create a virtual environment and install:

```bash title="pyvenv"
cd client
python -m venv .venv
# macOS/Linux
source .venv/bin/activate
# Windows (PowerShell)
# .venv\\Scripts\\Activate.ps1

pip install -r requirements.txt   # if present
pip install -e .                  # editable install for local dev
```

2. Run tests:

```bash title="commands.sh"
pytest -q
```

3. Linting / formatting:

```bash title="commands.sh"
ruff check .
black --check .
```

## Docker compose (optional / recommended for services)

To bring up dependent services (Postgres, Redis, etc.):

```bash title="commands.sh"
docker compose up --build
# or
docker compose up -d
```

When using Docker for the server, ensure environment variables are set (via `.env` or Compose overrides).

## Running the full system for development

A typical dev flow:

- Start Docker services (database, redis):
  `docker compose up -d`
- Run backend (Go) locally so it reloads on change, or run inside a container.
- Run frontend dev server and connect to the backend API via configured env vars.
- Activate Python client virtualenv and run `ckc` commands against the local server.

If the repo provides a `Makefile` with `make dev`, prefer that.

## Debugging

- Go: use Delve (`dlv`) and VS Code Go extension to set breakpoints (launch config pointing at `./cmd/server`).
- Python: use VS Code Python extension and the `pylance` language server; use the `python` debugger.
- Frontend: browser devtools and source-maps; VS Code debugging for Chrome/Edge.

Add logging environment variables (e.g., `LOG_LEVEL=debug`, `COOKIEFARM_DEBUG=true`) if supported.

## Common issues & troubleshooting

- "module not found" in Go: run `go mod tidy` and ensure `GOMOD` is at repo root.
- Frontend network issues (CORS / API base URL): verify `VITE_API_URL` / `REACT_APP_API_URL` (or relevant env key).
- Python dependency conflicts: ensure virtualenv activated and `pip install -e .` was used.
- Database migrations: check if the repo includes migrations under `migrations/` and run the migration tool used by the project (check docs).
- Ports already in use: verify server and frontend ports and adjust `.env` before starting.
